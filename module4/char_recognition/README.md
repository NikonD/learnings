# Как работает проект распознавания цифр

Проект учит нейросеть определять цифры 0–9 по изображению. Используются датасет MNIST (рукописные цифры) и свёрточная сеть на TensorFlow/Keras.

---

## Структура проекта

| Файл | Назначение |
|------|------------|
| `model.py` | Описание архитектуры нейросети |
| `train.py` | Обучение на MNIST и сохранение модели |
| `predict.py` | Загрузка своей картинки и предсказание цифры |
| `model.keras` | Сохранённая модель (появляется после `train.py`) |

---

## 1. Модель (`model.py`)

Функция `build_model()` собирает **свёрточную сеть (CNN)**:

1. **Conv2D(32)** — первый свёрточный слой: 32 фильтра 3×3, ищут простые признаки (линии, углы). Вход: изображение 28×28×1 (высота × ширина × один канал — оттенки серого).
2. **MaxPooling2D(2×2)** — уменьшение карты в 2 раза, снижение размера и шума.
3. **Conv2D(64)** — второй свёрточный слой: 64 фильтра, более сложные признаки.
4. **MaxPooling2D(2×2)** — снова сжатие.
5. **Flatten** — преобразование карты признаков в один вектор.
6. **Dense(128)** — полносвязный слой из 128 нейронов.
7. **Dropout(0.25)** — отключение 25% нейронов при обучении, чтобы уменьшить переобучение.
8. **Dense(10, softmax)** — выход: 10 вероятностей (по одной на каждую цифру 0–9).

**Компиляция:** оптимизатор `adam`, функция потерь `sparse_categorical_crossentropy` (многоклассовая классификация), метрика — точность (`accuracy`).

---

## 2. Обучение (`train.py`)

1. **Загрузка данных** — `tf.keras.datasets.mnist.load_data()` возвращает обучающую и тестовую выборки. При первом запуске MNIST скачивается автоматически.
2. **Подготовка:**
   - Пиксели нормализуются из диапазона 0–255 в 0–1 (`/ 255.0`).
   - К массиву добавляется размерность канала: (28, 28) → (28, 28, 1), так как модель ожидает формат (высота, ширина, каналы).
3. **Обучение** — `model.fit()` на 5 эпох, батч 128. На каждой эпохе сеть проходит по обучающим данным и подстраивает веса; на тесте считается точность.
4. **Сохранение** — обученная модель записывается в файл `model.keras` в папке проекта.

Без запуска `train.py` файла `model.keras` не будет — тогда `predict.py` выдаст сообщение, что нужно сначала обучить модель.

---

## 3. Предсказание (`predict.py`)

1. **Загрузка модели** — из файла `model.keras` (если его нет, скрипт завершается с подсказкой запустить `train.py`).
2. **Подготовка картинки** (`load_and_prepare_image`):
   - Открытие файла и перевод в оттенки серого (один канал).
   - Если средняя яркость > 127, изображение инвертируется (в MNIST цифра светлая на тёмном фоне).
   - Масштабирование до 28×28 (если размер другой).
   - Нормализация в 0–1 и добавление размерностей под формат модели: (1, 28, 28, 1) — один пример, высота, ширина, канал.
3. **Предсказание** — `model.predict(x)` возвращает вектор из 10 вероятностей; `argmax` даёт номер класса (0–9), он и выводится как распознанная цифра.

**Запуск из командной строки:**  
`python predict.py путь/к/картинке.png`

---

## Порядок работы

1. Установить зависимости: `pip install -r requirements.txt`
2. Обучить модель: `python train.py`
3. Распознать цифру на своей картинке: `python predict.py img/digit.png`

Картинка может быть любого размера (будет приведена к 28×28) и с тёмной или светлой цифрой — скрипт сам инвертирует при светлом фоне.
