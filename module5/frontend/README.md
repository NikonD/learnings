# Фронтенд к Company API (учебный модуль)

Два варианта клиента для бэкенда **module3/company_api_with_orm** (FastAPI + PostgreSQL):

| Папка       | Стек           | Для кого |
|------------|----------------|----------|
| **native** | HTML, CSS, JS  | Изучение работы с API без фреймворков |
| **framework** | React (Vite) | Изучение React и запросов к API |

## Как запускать

### 1. Запустить бэкенд

Из корня репозитория:

```bash
cd module3/company_api_with_orm
docker-compose up -d
pip install -r requirements.txt
python main.py
```

API будет доступен на **http://localhost:8000**. Документация: http://localhost:8000/docs

### 2. Запустить фронтенд

**Нативный (HTML/CSS/JS):**

- Открыть `native/index.html` через **Live Server** (VS Code) или любой другой статический сервер на порту 5500,  
  **либо** из папки `native` выполнить: `npx serve -l 5500`
- В браузере: http://localhost:5500

**React:**

```bash
cd module5/frontend/framework
npm install
npm run dev
```

В браузере откроется адрес (обычно http://localhost:5173)

## Что умеют оба фронта

- **Компании**: список, создание, редактирование, удаление
- **Должности**: список (с названием компании), создание, редактирование, удаление
- **Пользователи**: список, создание, редактирование, удаление (с привязкой к должностям)

Бэкенд должен быть запущен, иначе запросы будут падать с ошибкой сети.
